# Quickstart Guide

Welcome to ECHO — a local-first cognitive architecture designed to safely mediate LLM intent through deterministic workflows.

## What ECHO Is
ECHO is a seven-layer cognitive exoskeleton that:
- stabilizes identity
- enforces direction
- preserves human agency
- ensures deterministic execution
- prevents drift and hallucination

## How It Works
1. You provide intent.
2. The Interpreter aligns it with Doctrine.
3. State provides continuity.
4. HUD highlights what matters.
5. Execution turns intent into structured workflows.
6. Integrity ensures safety and coherence.
7. Evolution refines the system over time.

## Why Local-First?
- Privacy
- Determinism
- Stability
- Offline capability
- Human sovereignty

## How to Explore the Repo
- Start with the Preface
- Read the Architecture Diagram
- Explore each layer folder
- Review the Runtime Model

ECHO is designed to be understood from the outside without exposing internal logic.
