FAQ — ECHO Architecture
What is ECHO?
ECHO is a local‑first cognitive architecture designed to safely mediate LLM intent through a structured, deterministic, seven‑layer exoskeleton.
It strengthens human cognition without introducing autonomy, drift, or hidden state.

Does ECHO act autonomously?
No.
ECHO never initiates actions, goals, or tasks on its own.
It is explicitly designed to amplify human agency, not replace it.

ECHO proposes structure — the human decides.

Why seven layers?
Each layer isolates a specific cognitive responsibility:

Doctrine → identity & values

Interpreter → stability & alignment

State → continuity

HUD → salience & filtering

Execution → structured planning

Integrity → safety & coherence

Evolution → long‑term refinement

This separation prevents entanglement, drift, and unintended behavior.

Is the internal logic open‑source?
No.
Only the architecture, philosophy, and documentation are public.

The following remain private until patent filing:

kernel

doctrine text

interpreter logic

state structures

workflow internals

safety scaffolding

The repo exposes the shape, not the mechanism.

Can ECHO run offline?
Yes.
ECHO is intentionally designed to run entirely locally:

no cloud dependency

no external logging

no remote identity drift

no network requirement

Local‑first is a core design principle, not an optional mode.

Does ECHO replace human judgment?
Never.
ECHO is a cognitive exoskeleton, not an agent or decision‑maker.

It:

clarifies

structures

stabilizes

filters

decomposes

safeguards

…but the human remains the source of direction, authority, and judgment.

Is ECHO an agent framework?
No.
ECHO does not pursue goals, schedule tasks, or act independently.
It provides deterministic planning, not autonomous execution.

Does ECHO learn from user data?
No.
ECHO does not perform autonomous learning, fine‑tuning, or adaptation.
The Evolution Layer refines the architecture, not the user’s information.

What models does ECHO require?
Any local LLM can be used, as long as:

it is pinned

it is stable

it does not change without user approval

ECHO is model‑agnostic by design.

Can I contribute?
Yes — contributions to documentation, diagrams, and discussion are welcome.
The kernel and internal logic are not open for contribution.

Why is the kernel private?
To protect:

IP

safety mechanisms

identity stabilization logic

deterministic execution constraints

doctrine interpretation rules

The architecture is public; the mechanism is not.

Is ECHO production‑ready?
ECHO is architecture‑complete and kernel‑private, with ongoing refinement.
It is stable as a conceptual framework and evolving as a cognitive system.

refinement

long‑term evolution

ECHO evolves slowly and intentionally.
